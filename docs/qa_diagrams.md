# Diagram QA Checklist

Use this list whenever you add or update a drill diagram.

- [ ] **Field size & orientation** – matches the written dimensions (full vs half vs third) and arrows point toward the correct goal.
- [ ] **Player labels** – roles or numbers align with the drill description and are legible on desktop + mobile.
- [ ] **Equipment** – cones/goals appear where instructions mention them; no extra clutter.
- [ ] **Movement arrows** – clearly differentiate runs vs passes (color or stroke style) and avoid overlapping text.
- [ ] **Color contrast** – text/icons meet accessibility basics (white on green background, no neon clashes).
- [ ] **Export dimensions** – SVG/PNG fits within Streamlit cards without cropping or stretching.
- [ ] **File naming** – `assets/diagrams/<drill_id>.svg` so the loader can resolve it automatically.

Update this document whenever we tweak conventions (e.g., new icons, alternate color palettes).
